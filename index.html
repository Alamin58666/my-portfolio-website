My DevOps Journey

My DevOps Learning Journey: Tools, Cloud, Programming & Databases

Introduction

As a Computer Science student, I am actively building my skills in DevOps, cloud computing, web development, and programming. This blog shares my learning journey, the tools I use, and the technologies I am most interested in. My goal is to become a Full-Stack Developer first and then grow into a DevOps Engineer.

DevOps Tools: Docker & Git/GitHub

Docker is one of the most important DevOps tools I am learning. It helps package applications with all their dependencies into containers, making them easy to run on any system.

Why Docker is important:

  • Consistent environment across development and production
  • Faster deployment
  • Easy scalability
  • Works perfectly with CI/CD pipelines

I use Docker to containerize Node.js applications and understand how services communicate using containers.

Git & GitHub

Git and GitHub are essential for version control and team collaboration.

What I practice with Git/GitHub:

  • Repository management
  • Branching and merging
  • Pull requests
  • Issue tracking
  • Open-source collaboration

GitHub also helps me showcase my projects publicly and track my learning progress.

Cloud Computing: AWS & Terraform (Basics)

Amazon Web Services (AWS) - I am learning the fundamentals of cloud computing using AWS, focusing on the core services: EC2 (virtual servers), S3 (object storage), IAM (access management), and VPC (networking).

Terraform (Basic) - I am learning the basics to define cloud resources using code, create repeatable infrastructure, and reduce manual cloud setup.

Operating System: Linux (Ubuntu)

Linux is the backbone of DevOps. I mainly use Ubuntu Linux for:

  • File and directory management
  • User and permission control
  • Package installation
  • Basic shell scripting
  • Running servers and services
